# 08 — Claude Translator

Tests for `ajolote use claude` (generate) and Claude import during `ajolote init` / `ajolote sync`.

**Prerequisites for all tests:** Initialized project (`ajolote init`) with at least one rule file, unless stated otherwise.

---

## Generate

### CLAUDE-001 — CLAUDE.md uses @file references

**Prerequisites:** Config has `"rules": [".agents/rules/general.md"]`.  
**Steps:**
1. `ajolote use claude`
2. `cat CLAUDE.md`

**Expected result:** `CLAUDE.md` contains `@.agents/rules/general.md` (at-file reference, not inlined content). Also contains the ajolote generation header `> Generated by ajolote-ai...`.  
**Pass / Fail:** ☐

---

### CLAUDE-002 — settings.json contains project-scoped servers only

**Prerequisites:** Config has one project-scoped server (`"scope": "project"` or no scope field) and one user-scoped server (`"scope": "user"`).  
**Steps:**
1. `ajolote use claude`
2. `cat .claude/settings.json`

**Expected result:** `.claude/settings.json` contains only the project-scoped server. The user-scoped server is absent.  
**Pass / Fail:** ☐

---

### CLAUDE-003 — Commands written to .claude/commands/

**Prerequisites:** Config has commands in `.agents/commands/deploy.md` and `.agents/commands/review.md`.  
**Steps:**
1. `ajolote use claude`
2. `ls .claude/commands/`

**Expected result:** `.claude/commands/deploy.md`, `.claude/commands/review.md`, and `.claude/commands/ajolote-sync.md` all present. Content of each command file is preserved from source.  
**Pass / Fail:** ☐

---

### CLAUDE-004 — Scoped rules written to .claude/rules/

**Prerequisites:** Config has a scoped rule `{"name": "frontend", "globs": ["**/*.tsx"], "path": ".agents/rules/frontend.md"}`.  
**Steps:**
1. Create `.agents/rules/frontend.md` with content `# Frontend\nUse TypeScript.`
2. `ajolote use claude`
3. `cat .claude/rules/frontend.md`

**Expected result:** `.claude/rules/frontend.md` exists with frontmatter containing `description:` and `globs:` fields. The rule content follows.  
**Pass / Fail:** ☐

---

### CLAUDE-005 — Persona with `claude` block creates subagent file

**Prerequisites:** Config has a persona with a `claude` block:
```json
{
  "path": ".agents/personas/reviewer.md",
  "claude": {"model": "sonnet", "tools": ["Read", "Grep"]}
}
```
And `.agents/personas/reviewer.md` exists with content `# Code Reviewer\n\nReview code carefully.`  
**Steps:**
1. `ajolote use claude`
2. `cat .claude/agents/reviewer.md`

**Expected result:** `.claude/agents/reviewer.md` exists with YAML frontmatter: `name: Code Reviewer`, `model: claude-sonnet-4-6` (alias resolved), `tools: [Read, Grep]`.  
**Pass / Fail:** ☐

### CLAUDE-006 — Model alias "sonnet" resolves to full ID

**Prerequisites:** Persona with `"model": "sonnet"` in claude block.  
**Steps:**
1. `ajolote use claude`

**Expected result:** `model: claude-sonnet-4-6` (full model ID, not the shorthand).  
**Pass / Fail:** ☐

### CLAUDE-007 — Model alias "haiku" and "opus" resolve correctly

**Prerequisites:** Two personas — one with `"model": "haiku"`, one with `"model": "opus"`.  
**Steps:**
1. `ajolote use claude`
2. Inspect both subagent files.

**Expected result:** `haiku` → `claude-haiku-4-5-20251001`; `opus` → `claude-opus-4-6`.  
**Pass / Fail:** ☐

### CLAUDE-008 — Persona WITHOUT `claude` block does not create subagent

**Prerequisites:** Config has a persona as plain string `".agents/personas/simple.md"` (no claude block).  
**Steps:**
1. `ajolote use claude`
2. `ls .claude/agents/ 2>/dev/null || echo "no agents dir"`

**Expected result:** `.claude/agents/` directory does NOT exist (or is empty). No file created for the plain persona.  
**Pass / Fail:** ☐

### CLAUDE-009 — Persona description extracted from file when not explicit

**Prerequisites:** Persona with `claude` block but no `"description"` field; persona file first paragraph is `Review code carefully.`  
**Steps:**
1. `ajolote use claude`
2. `grep "description:" .claude/agents/*.md`

## Import

### CLAUDE-010 — Import MCP servers from settings.json

**Prerequisites:** Project with `.claude/settings.json` containing a server not in `.agents/config.json`.  
**Steps:**
1. `ajolote sync claude` (or `ajolote init` on a fresh project)
2. `cat .agents/config.json`

**Expected result:** The server from `settings.json` is now in `config.json`.  
**Pass / Fail:** ☐

---

### CLAUDE-011 — Import commands from .claude/commands/ (skips ajolote-sync.md)

**Prerequisites:** `.claude/commands/` contains `deploy.md` and `ajolote-sync.md`.  
**Steps:**
1. `ajolote sync claude`
2. `ls .agents/commands/`

**Expected result:** `.agents/commands/deploy.md` created. `ajolote-sync.md` is NOT imported (it is an ajolote internal command).  
**Pass / Fail:** ☐

---

### CLAUDE-012 — Import non-generated CLAUDE.md as rule file

**Prerequisites:** `CLAUDE.md` exists with hand-written content (no ajolote header).  
**Steps:**
1. `ajolote init` (fresh project)
2. `cat .agents/rules/general.md`

**Expected result:** `general.md` contains the content from `CLAUDE.md`.  
**Pass / Fail:** ☐

---

### CLAUDE-013 — Skip ajolote-generated CLAUDE.md during import

**Prerequisites:** `CLAUDE.md` starts with `> Generated by ajolote-ai from`.  
**Steps:**
1. `ajolote init` (fresh project)
2. `cat .agents/rules/general.md`

**Expected result:** `general.md` contains default boilerplate — not the contents of the generated `CLAUDE.md`.  
**Pass / Fail:** ☐

---

### CLAUDE-014 — User-scoped server written to ~/.claude.json

**Prerequisites:** Config has `"myserver": {"command": "npx", "args": ["pkg"], "scope": "user"}`.  
**Steps:**
1. `cp ~/.claude.json ~/.claude.json.bak 2>/dev/null || true` (backup)
2. `ajolote use claude`
3. `cat ~/.claude.json | grep "myserver"`
4. `cat .claude/settings.json | grep "myserver"`

**Expected result:** `"myserver"` appears in `~/.claude.json`. It does NOT appear in `.claude/settings.json`.  
**Pass / Fail:** ☐
